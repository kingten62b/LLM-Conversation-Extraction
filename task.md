### 0108 · 对话 → 结构化信息提取

###### 背景

客服主管每周（测试替换）需要了解所有客服对话的情况：用户问了什么、有没有解决、用户情绪怎么样……目前全靠逐条阅读，需要一个自动化工具。

###### 任务要求

> 1. 设计提取 Schema
>
> * 自己决定要从每条对话中提取哪些字段，说明每个字段的设计理由。
> * extract_example.md 有一条示例仅供参考，不代表标准答案，你需要自行设计完整 schema。
>
> 2. 实现提取工具
>
> * 对附件中 25 条对话自动提取，输出结构化 JSON。
> * 可用真实 LLM API，也可用 mock 模式模拟返回。
>
> 3. 处理边界情况：附件对话包含多种复杂情况（多诉求、转人工、话题切换、信息缺失等），在 README 中说明你的处理策略。
> 4. 验证准确性：人工抽检至少 5 条，报告准确率。
> 5. README：简要说明 schema 设计思路、任务拆解方式、AI 工具使用情况。

###### 交付物

请提供以下内容

1. **README** ：Schema 设计思路、任务拆解方式、边界处理策略、准确率、AI 工具使用情况
2. **开发工具截图** ：展示你的开发过程（IDE / Agent 工具 / 终端等）
3. **运行结果截图** ：提取结果示例、准确率等

> ###### 说明
>
> * 语言、框架不限
> * 鼓励使用 AI 工具辅助开发
